# QA Master Checklist

## Before Every Deploy

### Build
- [ ] Project builds successfully.
- [ ] TypeScript passes.
- [ ] Tests pass.
- [ ] No unused imports.
- [ ] No obvious console errors.

### Navigation
- [ ] Header links work.
- [ ] Footer links work.
- [ ] Home page links work.
- [ ] Tool cards link to correct pages.
- [ ] Blog links work.

### Mobile
- [ ] Home page looks good on mobile.
- [ ] Tools page looks good on mobile.
- [ ] Tool input is usable on mobile.
- [ ] Buttons are easy to tap.
- [ ] No horizontal scroll.

### Tool Functionality
- [ ] Valid input works.
- [ ] Invalid input shows helpful error.
- [ ] Empty input handled.
- [ ] Copy button works.
- [ ] Clear button works.
- [ ] Output is readable.

### SEO
- [ ] Each page has one H1.
- [ ] Each important page has metadata.
- [ ] URLs are clean.
- [ ] Tool pages have useful content.
- [ ] Internal links exist.
- [ ] Sitemap works.
- [ ] robots.txt works.

### Legal/Trust
- [ ] About page exists.
- [ ] Contact page exists.
- [ ] Privacy Policy exists.
- [ ] Terms page exists.

### AdSense Readiness
- [ ] No copied content.
- [ ] No prohibited content.
- [ ] No fake traffic.
- [ ] No ad code before approval.
