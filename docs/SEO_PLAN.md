# SEO Plan

## Strategy
Build useful free tools first, then write supporting articles for each tool. Each tool page should be indexable and useful even without the interactive UI.

### Developer Tools
- JSON formatter online
- README generator
- Git command helper
- Markdown previewer
- HTTP status code explainer
- SQL formatter
- Regex tester

### AI Coding

## Tool Page Template
Each tool page should include:

1. H1 with the main keyword.
2. Short explanation of the tool.
3. Interactive tool UI.
4. How to use section.
5. Example input and output.
6. Common mistakes.
7. FAQ section.
8. Related tools.
9. Related articles.

## Blog Article Template
Each article should include:

1. Clear title.
2. Short intro.
3. Practical example.
4. Step-by-step guide.
5. Common mistakes.
6. FAQ.
7. Internal links to related tools.
8. Internal links to related articles.

## Internal Linking Rules
- Home page links to top tools.
- Tools page links to every tool.
- Tool pages link to related blog posts.
- Blog posts link back to tools.
- Every article links to at least 2 internal pages.
- Every tool links to at least 2 related tools or articles.

## Content Quality Rules
- Do not publish thin pages.
- Do not copy content from other websites.
- Do not generate generic articles with no examples.
- Add practical examples.
- Add original explanations.
- Add FAQs based on real beginner questions.
